## User Story  
### สำหรับลูกค้า (Customer)
* **As a** ลูกค้า  
* **I want to** หยิบสินค้าใส่ตะกร้า, ยืนยันคำสั่งซื้อ และตรวจสอบสรุปรายการคำสั่งซื้อของฉัน  
* **So that** ฉันสามารถซื้อสินค้าที่ต้องการได้อย่างถูกต้องและติดตามสถานะคำสั่งซื้อได้  

### สำหรับผู้ดูแลระบบ (Admin)
* **As a** ผู้ดูแลระบบ  
* **I want to** ได้รับแจ้งเตือนออร์เดอร์ใหม่ และสามารถอัปเดตสถานะคำสั่งซื้อได้  
* **So that** สามารถจัดเตรียมและจัดส่งสินค้าให้ถึงมือลูกค้าได้อย่างถูกต้องตามกระบวนการ  

## Acceptance Criteria  
*These must be checked off before this task is considered done.* 
- [ ] **ระบบตะกร้าสินค้า (Customer):** ลูกค้าสามารถส่ง "ข้อมูลการหยิบใส่ตะกร้า" เพื่อเตรียมทำรายการได้
- [ ] **ระบบยืนยันคำสั่งซื้อ (Customer):** เมื่อลูกค้าส่ง "ข้อมูลยืนยันคำสั่งซื้อ" ระบบต้องดึง "ข้อมูลที่อยู่" จาก D1 ข้อมูลสมาชิก, "ข้อมูลรายการสินค้า" จาก D2 ข้อมูลสินค้า และ "ข้อมูลสต๊อก" จาก D3 ข้อมูลสต๊อก เพื่อประมวลผลคำสั่งซื้อ
- [ ] **ระบบสรุปรายการคำสั่งซื้อ (Customer):** ระบบต้องบันทึกข้อมูลลงใน D4 ข้อมูลคำสั่งซื้อ และส่ง "สรุปรายการคำสั่งซื้อ" กลับมาแสดงผลให้ลูกค้าตรวจสอบสถานะได้สำเร็จ
- [ ] **ระบบแจ้งเตือนแอดมิน (Admin):** ระบบต้องแสดงผล "แจ้งเตือนออร์เดอร์ใหม่" ให้ผู้ดูแลระบบทราบเมื่อมีรายการสั่งซื้อเข้ามาในระบบสำเร็จ
- [ ] **ระบบอัปเดตสถานะคำสั่งซื้อ (Admin):** ผู้ดูแลระบบสามารถส่ง "ข้อมูลอัปเดตสถานะสั่งซื้อ" เพื่ออัปเดตความคืบหน้า (เช่น กำลังจัดส่ง, สำเร็จ) ลงใน D4 ข้อมูลคำสั่งซื้อ ได้อย่างถูกต้อง

## Technical Specifications (SAs to fill out)  
* **Target Next.js Route/Component:** 
  * `/pages/cart` และ `/pages/checkout` (สำหรับลูกค้าทำรายการและดูสรุปออร์เดอร์)
  * `/pages/admin/orders` (สำหรับผู้ดูแลระบบดูแจ้งเตือนและอัปเดตสถานะสั่งซื้อ)
* **API Endpoint Required:** 
  * `POST /api/orders` (ยืนยันและบันทึกคำสั่งซื้อลง D4)
  * `GET /api/orders/summary/[id]` (ดึงสรุปรายการคำสั่งซื้อ)
  * `GET /api/admin/orders` (แอดมินดึงรายการและตรวจรับการแจ้งเตือนออร์เดอร์ใหม่)
  * `PATCH /api/admin/orders/[id]/status` (อัปเดตสถานะคำสั่งซื้อ)
* **MySQL Tables Impacted:** `Members` (D1), `Products` (D2), `Stock` (D3), `Orders` / `Order_Details` (D4)
